Buttons

Primary CTA Button

Our button definitions and look-and-feel come from Bootstrap, you can find the definitions and css settings in bootstrap.css.

All buttons use the parent .btn class in order to set some initial common styles and behaviors like border-color.

Primary buttons, or Call-to-Action buttons, are created with the .btn-primary class. Add that to your button definition as shown above. Our primary CTA buttons are a royal blue color.

Secondary buttons are used in forms when we absolutely cannot omit them. They are a light grey color, slightly smaller than the primary buttons, and are created by simply using the .btn class on its own. This is the default style for a button so there is no need to add additional clases.

We use orange to indicate help or support. Our Intercom "talk to us" button is orange, "more information" indicators on forms are orange, and any links to help documentation or other assistance is orange as well. You can create a support button with the class .btn-warning.


Primary buttons are used in dashboards or when they occur alone. Back buttons are used in forms (their height is the same as the input fields).

Buttons can be created using the button tag accompanied by the button class.

To make a Primary button just combine primaryBtn with the button class. To make a Back button use seconderyButton class.
